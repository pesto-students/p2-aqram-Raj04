Que.3.2) Create 3 functions call, apply and bind

  • Reusable code is very commonly used to solve many problem, we dont want writing the same piece of code again and again. Here comes the application of call, apply and bind.
  •  Say there are two objects holding addresses, and the function there to print all the address details. The issue we can see is there is a function common between them that is printing its content.
  • There are two ways to deal with the above situation, either we keep function in one address type object and borrow it when needed in another function or another way to keep a seperate function and use it when needed.

Call()

  • Below code is the demonstration on how to use call method

let addressObj1 = {
state: "West Bengal",
city: "Salt lake",
pincode: "742586",
getAddress: function () {
  console.log(
    `Address is State: ${this.state},City: ${this.city}, pincode: ${this.pincode}`
  );
}
};
let addressObj2 = {
state: "Bihar",
city: "Patna",
pincode: "712256"
};

addressObj1.getAddress.call(addressObj2);
O/P:
Address is State: Bihar,City: Patna, pincode: 712256

  • The existing createStack() is returning an original items array i.e stack in our case, push(), pop(). Point to be here noted we are returning original array that holds the items .
  • Thus we are giving the control to the caller to call and change the items array anytime in the code, and it will change on modifying. This shouldnt happen
  • To prevent any change in items array, we have to remove items array from returned object and keep it in the outer scope, so that it maintains abstraction and the user should only be concern of using the functionalities such as push, pop and print(added below).
  • Below code is the example of how we should prevent change in original array

function createStack() {
var items = new Array();

return {
  // items: [],
  push(item) {
    return items.push(item);
  },
  pop() {
    return items.pop();
  },
  printStackElements() {
    console.log(items);
  }
};
}

const stack = createStack();
stack.push(10);
stack.push(5);
stack.push(5);
stack.items = [100, 100]; // it doesn’t work


console.log(stack.pop());
stack.printStackElements();

 Create a memoize function that remembers previous inputs and stores them in cache so that itwon’t have to compute the same inputs more than once. The function will take an unspecifiednumber of integer inputs and a reducer method.

function add(a, b) {
return a + b;
}
//Create a method called memoize
function memoize(func) {
var cache = {};
return function () {
  var key = JSON.stringify(arguments);
  if (cache[key]) {
    return cache[key];
  } else {
    var val = func.apply(this, arguments);
    console.log(cache);
    cache[key] = val;
    return val;
  }
};
}
// such that:

const memoizeAdd = memoize(add);
